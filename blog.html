Week 7 - Firebase6/4/2016

This week we ventured into using Firebase to persistently store data. This was a pretty great jump in terms of creating websites that can have a life cycle longer than just the user's initial interaction with them (i.e. the games and apps that we created leading up to this week).

We used jQuery to capture user input from a form, and then implemented the Firebase object to post that data to Firebase. So as I can gather at this point, Firebase is basically like 'server as a service', which allows easy storage of data to be used, appended to, referenced, and so on over time.

One interesting challenge we solved this week was how to calculate the amount of time remaining before the arrival of a train, based on the frequency of the train and the current time. To help with this task, we learned about and used Moment.js, a great JS library for working with time. It allows for so many options in terms of how to display the time, how to help calculate time, etc. If you're looking for a resource to use for any time-related issues, I'd highly recommend it.

Week 6 - AJAX & APIs5/28/2016

This week we ventured into making AJAX calls to APIs using jQuery's $.ajax functionality, specifically the 'POST' method. During my self study, I had never really moved past creating something and putting it out there for people to see, and this was my first exposure to requesting data and getting a response. The JSON data that came bac kat first was intimidating, particularily in figuring out how to traverse it and get at what you need. When objects are nested inside of arrays, which are nested inside other arrays, and all part of an object, you've really got to do some detective work to get the specific category of data you need. Your code ends up looking something like:

var movieTitle = response.data[0].info.movie.title[0];

To make things easier, enter Postman. Using this app, you can get a much better view of the data structure and even paste in the query URLs to check out the data before you try to impliment it into your own app. It's super useful and I recommend it to any developer getting started with making API calls.

We also delved pretty deep into using data attributes in our DOM elements to help control their state. It seems like these are some pretty important and very useful properties for controlling elements. I'm going to read more about the aria attributes I've seen before because they seem pretty similar.

Week 4 - jQuery5/14/2016

This week we learned about jQuery, Javascript's favorite offshoot. I really enjoyed this week; jQuery makes writing seemingly complicated things in Javascript muuuuuuch easier. Instead of a bunch of 'document.getElementById()' statements, you can simply use jQuery's 'bling' ($) operator to select elements with a certain class or id. So '$(.nav-link).on("click", function() etc..." will select all elements with the class 'nav-link' and listen for a click event. Whoa. Super simple; super useful; and it really made me feel like I had much more control over my document, and therefore my overall application. I. Love. jQuery.

nb: It's always a good idea to include the following code at the beginning of your document: 

$(document).ready(function() {
   // stuff 
 }

The reason it's a good idea to include the above code is because you want the document (aka the DOM) to be fully loaded before your user starts interacting with it. If not, some odd things could possibly happen.

We also learned about the concept of 'scope'. This is very important when writing Javascript because it affects how data stored in variables can be used throughout the process. Basically, any variable declared inside a function can only be used inside that function, and all other areas outside of that function. In other words, the scope of a variable can bubble out, but not in. There is a great article here that explains scoping much better than I can at this point.

Week 2 - Heroku, More CSS, Bootstrap4/30/2016

This week we learned about the hosting site Heroku , which is a great way to publish your sites live on the internet. The site itself is designed for dynamic websites (which, we don't know any PHP yet), so our instructors helped us include two files which sort of 'hacks' Heroku to allow hosting of static sites. I believe this is how we'll be turning in our homework assignments from now on.

We also learned about the framework Bootstrap, which was created by ©Twitter. It's a very powerful and easy to use framework for HTML, CSS, and even some JS functionality. It puts mobile-responsiveness first, which is important in modern web trends because so many people access sites through their small devices (smartphone, tablet, etc.). With the framework, all you have to do is add classes to DOM elements to make them appear correctly and respond in certain ways when resizing the screen (much easier than writing your own @media queries!). The basic flow for a Bootstrap site is: 1) container; 2) row; 3) column. Following this pattern, it's pretty easy to layout your content correctly. One super important step when beginning to develop any project is to work out a wireframe and to do a sketch. This will help visualize how the site will look on the screen.

Our homework assignments this week were to create two portfolio sites, one using basic HTML and CSS, and one using a Bootstrap theme. The latter seems a bit like cheating, but I suppose it's a quick and easy way to get a site live on the web on which you can host your other projects. I think I'll follow this route in the future.

Week 1 - Course Intro4/23/2016

Here we go; first week of class is in the books! A lot of the things we learned in class this week I had already learned through self-study. One thing that was new and definitely valuable was learning some very basic command line commands, like 'pwd' for 'present working directory' (aka to see where you are in the terminal); 'cd' for 'change directory' to move around in the terminal; 'ls' for 'list' to see a list of files and directories in the current directory; 'touch' (which is a strange command word choice) for creating new files; 'mkdir' for 'make directory' to create a directory (or folder); 'rm' for 'remove' to delete a file (and, subsequently, 'rmdir' for 'remove directory' to delete a directory). All this was very new and welcome information. It really makes you feel like you know what you're doing when you use the command line to execute even simple tasks like the ones above.

We also learned about Git and GitHub this week and the very important, fundamental concept that Git ≠ GitHub. Git is a mysterious program used in the termial for version control (which, at this point...whatever that means, lol). GitHub is where you store your code online...using Git to push the code to GitHub. Yeah. It's confusing.

We also learned about the basic structure of an HTML document and some important CSS, including the box-model. Our TA, Holly, kept talking about 'Flexbox' in CSS, which sounds pretty cool and useful. I'll have to go back and read about that. One extremely helpful thing I learned about CSS this week is that when you set the property 'box-sizing: border-box', you can then mainpulate the 'padding', aka the space between the content and the edge of the box, and it will not make the box bigger. Super helpful when trying to perfect layout.
